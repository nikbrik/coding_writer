# Goal Loop Rules

- Work the task until the explicit goal is satisfied by evidence or a real blocker is reached.
- Prefer the smallest meaningful next step over broad rewrites.
- Verify changes with the minimal command, test, or inspection that proves progress.
- Re-review after verification. Do not call the task done while there are unresolved findings.
- Respect explicit approval gates, diff-before-write requirements, and user confirmation rules. Persistence must not bypass approvals.
- When a task has a formal goal file or goal-oriented protocol, read that first and treat it as the source of truth.
